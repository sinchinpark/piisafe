# Plan: Simplify KEK API — single `kek_keys` param

## Problem

Two params `kek_key` and `kek_keys` is confusing. Just use `kek_keys` — accept a single key or list.

## Changes

### 1. `python_pii/service.py` — simplify `__init__`

```python
def __init__(
    self,
    storage: PIIStorageBackend,
    kek_keys: Optional[List[bytes]] = None,
):
```

- `kek_keys` accepts `bytes` (single key) or `List[bytes]` (multiple keys)
- Normalize: `if isinstance(kek_keys, bytes): kek_keys = [kek_keys]`
- Env var fallback: `FERNET_KEYS` (comma-separated) > `FERNET_KEY` (single)

### 2. Update all callers

| File | Old | New |
|------|-----|-----|
| `tests/test_service.py` | `kek_key=key` | `kek_keys=key` |
| `tests/adapters/conftest.py` | `kek_key=fernet_key` | `kek_keys=fernet_key` |
| `tests/test_backends.py` | `kek_key=Fernet.generate_key()` | `kek_keys=Fernet.generate_key()` |

### 3. Update README examples

All `kek_key=` → `kek_keys=` in Quick Start and Configuration sections.

### 4. Update docstrings

Remove `kek_key` references, document single key as `kek_keys=key`.
